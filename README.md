# 📄 Document Chatbot - RAG Application

A powerful document-based chatbot that uses **Retrieval-Augmented Generation (RAG)** to answer questions about your documents. This application runs locally using open-source models and doesn't require API keys.

## 🌟 Features

- **Local LLM**: Uses Ollama with the Gemma 2B model for completely offline operation
- **Document Processing**: Automatically loads and processes `.docx` files from your documents folder
- **Smart Retrieval**: Uses FAISS vector database with HuggingFace embeddings to find relevant document sections
- **Source Citations**: Shows which documents were used to generate each answer
- **User-Friendly Interface**: Built with Streamlit for an intuitive web-based UI

## 📋 Prerequisites

Before setting up this project, make sure you have:

- **Python 3.8+** installed on your system
- **Ollama** installed and running (download from [ollama.ai](https://ollama.ai))
- **Gemma 2B Model** pulled in Ollama: `ollama pull gemma:2b`
- **pip** (Python package manager)

## 🚀 Setup Instructions

### Step 1: Clone/Navigate to the Project
```bash
cd PythonChatBot
```

### Step 2: Create a Virtual Environment (Optional but Recommended)
```bash
# On Windows
python -m venv venv
venv\Scripts\activate

# On macOS/Linux
python -m venv venv
source venv/bin/activate
```

### Step 3: Install Dependencies
```bash
pip install -r requirements.txt
```

### Step 4: Prepare Your Documents
- Place your `.docx` files in the `documents/` folder
- The chatbot will automatically load all Word documents from this directory
- You can organize documents in subfolders within `documents/`

### Step 5: Ensure Ollama is Running
```bash
# Make sure Ollama is running (typically runs on localhost:11434)
ollama serve
```

## ▶️ Running the Application

Once setup is complete, start the chatbot:

```bash
streamlit run app.py
```

The application will open in your web browser at `http://localhost:8501`

## 📖 How to Use

1. **Wait for Pipeline**: The app will display a loading message as it initializes the RAG pipeline
2. **Ask Questions**: Type your question in the text input field about your documents
3. **Get Answers**: The chatbot retrieves relevant document sections and generates answers using the Gemma model
4. **View Sources**: Click on the "Sources" expandable section to see which documents were used

## 📝 Configuration

Edit `config.py` to customize your setup:

```python
DOC_PATH = "./documents"      # Path to your documents folder
MODEL_NAME = "gpt-4"          # Reference configuration (currently using Ollama)
```

### Changing the LLM Model

To use a different Ollama model, modify `rag_pipeline.py`:

```python
llm = Ollama(model="llama2")  # Change to another available Ollama model
```

Available Ollama models:
- `gemma:2b` (default - fast, lightweight)
- `llama2` (more capable, slower)
- `mistral` (good balance)

### Adjusting Embeddings

To use different embeddings, modify `rag_pipeline.py`:

```python
embeddings = HuggingFaceEmbeddings(model_name="sentence-transformers/all-mpnet-base-v2")
```

## 🔧 Components Explained

### RAG Pipeline (`rag_pipeline.py`)

**Document Loading**: 
- Recursively loads all `.docx` files from the documents folder
- Uses UnstructuredWordDocumentLoader for proper text extraction

**Text Chunking**:
- Splits documents into 500-character chunks with 100-character overlap
- Ensures context is preserved across chunks

**Vector Store**:
- Uses FAISS (Facebook AI Similarity Search) for fast retrieval
- Embeddings generated by HuggingFace's `all-MiniLM-L6-v2` model

**QA Chain**:
- RetrievalQA chain fetches the top 3 most relevant chunks
- Ollama Gemma 2B model generates answers based on retrieved context

### Streamlit Interface (`app.py`)

- Simple, clean UI for document-based Q&A
- Real-time response generation with loading indicator
- Source document attribution in expandable section
- Error handling for graceful failure messages

## 🐛 Troubleshooting

### "Connection refused" or Ollama not found
- **Solution**: Make sure Ollama is running. Start it with `ollama serve` in another terminal

### "No documents found"
- **Solution**: Ensure you have `.docx` files in the `documents/` folder
- **Tip**: Check file permissions and ensure the path is correct

### Slow response time on first run
- **Solution**: The vectorstore is being created for the first time. Subsequent queries will be faster
- **Tip**: Larger document sets will take longer to index

### Out of memory errors
- **Solution**: Reduce `chunk_size` in `rag_pipeline.py` or use fewer/smaller documents
- **Tip**: Gemma 2B is designed to be lightweight

### Module not found errors
- **Solution**: Reinstall dependencies with `pip install -r requirements.txt`
- **Tip**: Ensure your virtual environment is activated

## ⚙️ Performance Tips

1. **First Run**: Document processing takes time. The vectorstore is cached after creation
2. **Chunk Size**: Smaller chunks = more retrieval accuracy, but slower processing
3. **Model Size**: Gemma 2B is lightweight; use larger models for better quality
4. **Document Quality**: Well-formatted documents with clear content work best
